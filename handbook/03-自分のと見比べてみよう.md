# 🪞 03 — 自分のと見比べてみよう

Stage 4 で作ったあなたの `my-profile` と、`pull-practice/my-profile`（このリポジトリの完成形）を **見比べる**ステップです。

⭐ **目的:** 「ちゃんと出来てた？」を確認するためではなく、**「AI に頼むときの引き出しを増やす」** ためです。

---

## ⚠️ 先に伝えたいこと

**「自分のと違う！ダメだった！」と思わなくて大丈夫**です。

- ✅ プログラミングに **「唯一の正解」はありません**。同じ自己紹介ページを10人が作れば10通りになります
- ✅ お手本と違っても、**あなたのコードが動いていればそれで合格**
- ⭐ **違いに気づくこと自体が学び**。違いを比較できる目を育てるのがこのステップ

---

## 💬 Claude Code に頼む3つのプロンプト

3つあります。**1つだけでもOK**、3つ全部やれば理解が深まります。

### プロンプト①「3つだけ違いを教えて」

```
私の my-profile/src/App.tsx と、pull-practice/my-profile/src/App.tsx を見比べて、
大きな違いを3つだけ、初心者向けの言葉で教えて。
正解・不正解の話ではなく「こういう書き方もあるんだ」という発見として。
```

⭐ **得られるもの:** 「あ、自分のはこう書いてた、お手本はこうなんだ」という気づき。

---

### プロンプト②「真似したいところを取り入れて」

```
pull-practice/my-profile/src/App.tsx の書き方を見て、
私の my-profile/src/App.tsx に取り入れたら良くなりそうな点を1つだけ提案して。
私のコードを実際に書き換えるところまでやって。
変えた箇所は何で、なぜ良くなるのかも教えて。
```

⭐ **得られるもの:** **動いているコードに、お手本の良さを少しだけ足す**体験。これがプロの学び方です。

---

### プロンプト③「初心者にもわかる例えで」

```
pull-practice/my-profile/src/App.tsx の中身を、
プログラミング初心者にもわかる例え話だけで説明して。
コードは見せず、「これは○○みたいなもの」という形式で。
```

⭐ **得られるもの:** コードを「概念」に翻訳する力。**他人に説明できれば、それは本当に理解できた証拠**です。

---

## 🎯 比較するときに見るとよい5つの観点

| 観点 | 何を見る |
|---|---|
| 🎨 **色** | 青系（`blue-*`）になっているか、文字色とのコントラスト |
| 🃏 **Card の構造** | `<Card>` `<CardHeader>` `<CardContent>` を使っているか |
| 🔣 **アイコン** | `lucide-react` のアイコンが使われているか、なくても可 |
| 📐 **配置** | 中央寄せか、余白の取り方 |
| 📝 **項目** | 名前・役割・好きなこと・今学んでいることが揃っているか |

---

## 🤔 「自分のは全然違う」と感じたら

⭐ **それはむしろチャンス**です。

Claude Code にこう聞いてください:

```
私の my-profile/src/App.tsx を、
pull-practice/my-profile/src/App.tsx と「同じ作りに揃えて」書き直して。
ただし、私の名前と内容（[あなたの実際の名前・役割・趣味]）はそのままで。
```

⭐ AI が **あなたのデータはそのままに、構造だけお手本に揃えて**くれます。これで「お手本と同じ作りのプロフィールページ」が手に入ります。

---

## ✨ 最後に

このステップで一番大事なのは、**コードの良し悪しではなく**:

> **「違いがある」→「AIに頼んで揃えられる」→「やってみる」**

という **AI と一緒に改善するループ**を1回体験することです。

⭐ Stage 6 以降では、この **「お手本と比べて、AIに頼んで近づける」** をたくさん使います。今ここで1回やっておけば、これからずっと使えます。
