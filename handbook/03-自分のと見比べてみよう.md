# 🪞 03 — 自分のと見比べてみよう

Stage 4 で作ったあなたの `my-profile` と、`pull-practice/my-profile`（このリポジトリの完成形）を **見比べる**ステップです。

⭐ **目的:** 「ちゃんと出来てた？」を確認するためではなく、**「AI に頼むときの引き出しを増やす」** ためです。

---

## ⚠️ 先に伝えたいこと

**「自分のと違いすぎる！ダメだった！」と思わないで大丈夫**です。

- ✅ Stage 4 では「Card で自己紹介ページを表示する」がゴール。**それは達成できているはず**
- ✅ ここにあるのは「Stage 1〜5 全部が積み上がった先にある完成イメージ」。今すぐ全部書ける必要はありません
- ⭐ **違いに気づくこと自体が学び**。「こんな書き方もあるんだ」を吸収する場として使ってください

---

## 💬 Claude Code に頼む3つのプロンプト

3つあります。**1つだけでもOK**、3つ全部やれば理解が深まります。

### プロンプト①「3つだけ違いを教えて」

```
私の my-profile/src/App.tsx と、pull-practice/my-profile/src/App.tsx を見比べて、
大きな違いを3つだけ、初心者向けの言葉で教えて。
正解・不正解の話ではなく「こういう書き方もあるんだ」という発見として。
```

⭐ **得られるもの:** 「自分のはこう、お手本はこう」という気づき。データを別ファイルにする工夫や、複数 Card で構成を分ける書き方が見えてきます。

---

### プロンプト②「真似したいところを1つだけ取り入れて」

```
pull-practice/my-profile を読んで、
私の my-profile に「1つだけ」取り入れたら良くなりそうな点を提案して。
私のコードを実際に書き換えるところまでやって。
変えた箇所と、なぜ良くなるのかも教えて。
私の名前と内容（[あなたの実際の名前・役職・好きなこと]）はそのままで。
```

⭐ **得られるもの:** **動いているコードに、お手本の良さを少しだけ足す**体験。これがプロの学び方です。

---

### プロンプト③「データだけ自分のに置き換えて」

```
pull-practice/my-profile/src/data/profile.ts の構造をそのまま使って、
私のプロフィールに書き換えた profile.ts を作って。

私の情報:
- 名前: [あなたの名前]
- 役職: [例: 経理 / 看護師 / 営業 など]
- 経歴: [覚えている範囲で]
- スキル: [PCで使えるツール何でも。Excel, PowerPoint, etc.]
- 今学んでいること: Claude Code研修 と [他のあれば]
- 連絡先: [メールアドレス or ダミーで]

それを使えるように、私の my-profile/src の中にデータファイルを作って、App.tsx を書き換えて。
```

⭐ **得られるもの:** **あなた専用の職務経歴書ページ**が一気に手に入ります。Stage 6 以降のポートフォリオの土台にもなる。

---

## 🎯 比較するときに見るとよい7つの観点

| 観点 | 何を見る |
|---|---|
| 🏗 **構成** | あなたのは1個のCardだけ？ お手本は何個のセクションに分かれているか |
| 📦 **データの置き場所** | あなたのは App.tsx に直書き？ お手本はどうしているか（profile.ts に分離） |
| 🃏 **Card の使い方** | 1枚 vs 複数枚。それぞれにヘッダー＆中身がある構成 |
| 🏷 **Badge の有無** | スキルや成果指標をタグで見せる工夫があるか |
| 🗂 **タイムライン** | 経歴を年表で並べる発想があるか |
| 🎨 **色とアクセント** | グラデーション帯・アクセントカラー・hover で変わる感じ |
| 📱 **モバイル対応** | 375px に縮めても崩れない仕掛け（`grid-cols-1 lg:grid-cols-[...]`） |

---

## 🤔 「自分のは全然違う」と感じたら

⭐ **それはむしろチャンス**です。

Claude Code にこう聞いてください:

```
私の my-profile/src/App.tsx を、
pull-practice/my-profile の構成（左サイドにプロフィール/連絡先/スキル、右にAbout/経歴/作品/学習中/学歴）に揃えて書き直して。
ただし、私の名前と内容（[あなたの実際の名前・役職・経歴]）はそのままで。
profile.ts のようなデータファイルを分離する形にして。
```

⭐ AI が **あなたのデータはそのままに、構造だけお手本に揃えて**くれます。これで「お手本と同じ作りのプロフィールページ」が手に入ります。

---

## ✨ 最後に

このステップで一番大事なのは、**コードの良し悪しではなく**:

> **「違いがある」→「AIに頼んで揃えられる」→「やってみる」**

という **AI と一緒に改善するループ**を1回体験することです。

⭐ Stage 6 以降では、この **「お手本と比べて、AIに頼んで近づける」** をたくさん使います。今ここで1回やっておけば、これからずっと使えます。
