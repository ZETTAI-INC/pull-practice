# 📘 05 — 用語ミニ辞典

Stage 1〜5 で出てきた英単語を、**初心者向けの例え**で全部おさらいします。

⭐ **覚えなくて大丈夫。** 「なんとなく分かる」程度になればOK。詰まったらここに戻ってきてください。

---

## 🛠 ツール系

### Antigravity（アンチグラビティ）

**何？** コードを書くためのエディタ（メモ帳の進化版）。Claude Code を呼び出せる窓口でもある。

**例えると?** AIアシスタント付きの作業デスク。

> ※ Stage 1 で受講

---

### Claude Code（クロードコード）

**何？** ターミナルで動く AI。`claude1` と打つと起動する。

**例えると?** 隣に座ってる先輩エンジニア。日本語で頼めば手を動かしてくれる。

> ※ Stage 1 から毎ステージ使用

---

### ターミナル

**何？** 文字でコマンドを打つ画面。Antigravity の下部にある黒い画面。

**例えると?** PCに直接話しかけるトランシーバー。

> ※ Stage 2 で受講

---

### GitHub（ギットハブ）

**何？** コードをネット上に保存するサービス。

**例えると?** コード版の Google ドライブ。**世界中どこからでも取り出せる金庫**。

> ※ Stage 5 で受講

---

## 📁 プロジェクト系

### Vite（ヴィート）

**何？** プロジェクトの骨組みを一瞬で作るツール。

**例えると?** 家を建てるときの **基礎工事の職人**。土台を作って引き渡してくれる。

> ※ Stage 4 で受講

---

### React（リアクト）

**何？** 画面を「部品」で組み立てる仕組み。Webアプリの世界標準。

**例えると?** レゴブロック。部品（Card, Button...）を組み合わせて画面を作る。

> ※ Stage 4 で受講

---

### TypeScript（タイプスクリプト）

**何？** JavaScript に「型」を足した言語。間違いを早めに教えてくれる。

**例えると?** **誤字を指摘してくれる赤ペン先生**。書きながら間違いに気づける。

> ※ Stage 4 で受講

---

### Tailwind CSS（テイルウィンドシーエスエス）

**何？** 「色」「余白」「文字サイズ」をクラス名1つで指定できる仕組み。

**例えると?** ファッションのコーディネートカタログ。「青系」と書くだけで青になる。

> ※ Stage 4 で受講

---

### shadcn/ui（シャドシーエヌ）

**何？** Card・Button・Table などのプロ品質の UI 部品集。

**例えると?** **無印良品のような既製品の家具屋**。「Card ください」と頼めばすぐ使える。

> ※ Stage 4 で受講

---

### Card（カード）

**何？** 情報を1枚のカード形に整える UI 部品。

**例えると?** 名刺の用紙。1枚に名前・役割・情報を整えて並べられる。

> ※ Stage 4 で受講

---

### localhost（ローカルホスト）

**何？** 自分のパソコンだけでアプリを動かしている状態。

**例えると?** **自分の部屋だけで上映している試写会**。ネットには公開されていない。

> ※ Stage 4 で受講

---

## 💾 Git / GitHub 系

### Git（ギット）

**何？** コードの変更履歴を管理する仕組み。

**例えると?** **タイムマシン付きの記録ノート**。いつでも過去の状態に戻れる。

> ※ Stage 5 で受講

---

### リポジトリ

**何？** GitHub 上の「1つのプロジェクトを丸ごと置く場所」。

**例えると?** Google ドライブの **共有フォルダ**。1つのアプリにつき1つ作る。

> ※ Stage 5 で受講

---

### commit（コミット）

**何？** 今の状態を「1つの記録」として保存すること。

**例えると?** **スマホで写真を撮る瞬間**。「この時点」のスナップショット。

> ※ Stage 5 で受講

---

### push（プッシュ）

**何？** ローカル（自分のPC）の commit を GitHub に送ること。

**例えると?** 撮った写真を **クラウドにアップロード**する操作。

> ※ Stage 5 で受講

---

### clone（クローン）

**何？** GitHub 上のリポジトリを、初めて自分のPCに丸ごとコピーすること。

**例えると?** 共有フォルダから **初回ダウンロード**。まだローカルに何もない時に使う。

> ※ Stage 5 で受講

---

### pull（プル）

**何？** すでに自分のPCにあるリポジトリを、最新状態に更新すること。

**例えると?** 共有フォルダの **同期ボタン**を押す感覚。差分だけ降りてくる。

⭐ **Stage 5 で実際にやったこと。このハンドブックも pull で降りてきた。**

> ※ Stage 5 で受講

---

### .gitignore（ドットギットイグノア）

**何？** 「GitHub に上げたくないファイル」を書くリスト。

**例えると?** **「これはアップしないで」と書いた付箋**。`.env` を守るために必須。

> ※ Stage 5 で受講

---

### .env（ドットエンブ）

**何？** パスワードやAPIキーを入れるファイル。

**例えると?** **金庫の鍵を書いた紙**。絶対に GitHub にアップしてはいけない。

> ※ Stage 5 で受講

---

### タグ（tag）

**何？** コミット履歴の特定の地点に貼る「しおり」。

**例えると?** **本のしおり**。「ここから始めて」と印を付けられる。

> ※ Stage 5 で受講（`start` タグから clone した）

---

## 🖥 コマンド系

### `npm install`

**何？** プロジェクトで使うライブラリ（部品）をまとめてダウンロード。

**例えると?** 料理の前に **材料を全部仕入れる**。

---

### `npm run dev`

**何？** 開発サーバーを起動して、ブラウザで動作確認できる状態にする。

**例えると?** **試食用のキッチンに火をつける**。ブラウザで `http://localhost:5173` を開けるようになる。

---

### `git add .`

**何？** 「次の commit に含めるファイル」を選ぶ。`.` は「全部」を意味する。

---

### `git commit -m "メッセージ"`

**何？** 選んだファイルを「1つの記録」にする。

---

### `git push`

**何？** ローカルの commit を GitHub に送る。

---

### `git clone -b [タグ名] [URL]`

**何？** 指定したタグの状態でリポジトリを取り込む。

**例:** `git clone -b start https://github.com/.../pull-practice.git`

---

### `git pull origin main`

**何？** GitHub の `main` ブランチの最新を、今いるフォルダに取り込む。

⭐ **このハンドブックを取り込んだコマンド。**
