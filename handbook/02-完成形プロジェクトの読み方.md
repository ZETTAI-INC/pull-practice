## ⭐ 読む順番（推奨）

### 1️⃣ まず `src/App.tsx`（5分）

**ここに自己紹介ページの中身が全部書いてあります。**

注目してほしい点:

| 何を見る | コードのどこ |
|---|---|
| **Card の構造** | `<Card>`、`<CardHeader>`、`<CardContent>` の3段ネスト |
| **アイコン** | `<User />`、`<Briefcase />` などが `lucide-react` から import されている |
| **色** | `bg-blue-50`、`text-blue-700` など `bg-` `text-` で始まる Tailwind クラス |
| **項目** | 名前・役割・好きなこと・今学んでいること の4つ |

### 2️⃣ 次に `package.json`（2分）

`"dependencies"` の中を眺めてください。これが「このアプリで使ったライブラリ一覧」です。

| ライブラリ | 何をしてくれる人？ |
|---|---|
| `react` | 画面を組み立てる本体 |
| `react-dom` | React と ブラウザをつなぐ |
| `lucide-react` | アイコンを提供 |
| `@radix-ui/react-slot` | shadcn/ui の内部で使う部品 |
| `class-variance-authority` | クラス名を場合分けする |
| `clsx` `tailwind-merge` | クラス名を合成する |

⭐ **覚えなくて大丈夫。** 「AI がこういう部品たちを組み合わせてくれた」と分かれば十分です。

### 3️⃣ 余裕があれば `.gitignore`（1分）

**`.env` と `.env.*` の行**を探してください。これが **「.env を GitHub に上げないための設定」** です。

Stage 5 で「.env が GitHub にあがっていないか確認」したのは、この `.gitignore` が効いているからです。

## ❌ 読まなくていいもの

- `vite.config.ts` / `tsconfig.json` / `tailwind.config.js` / `postcss.config.js`
- `card.tsx` の中身（shadcn/ui の定型コードで、自分で書くものではない）
- `main.tsx`（4行だけ。App を呼ぶだけのファイル）
- `utils.ts`（cn という1つの関数だけ）

⭐ **「設定ファイルは AI に任せる」** が鉄則。今は読まなくていいです。
