## ⭐ 読む順番（推奨）

### 1️⃣ まず `src/data/profile.ts`（5分）

**ここに人物データが全部書いてあります。** ⭐ 「自分のプロフィールに置き換えるなら、このファイルだけ書き換えればいい」状態になっています。

注目してほしい点:

| 何を見る | 何が学べる |
|---|---|
| `profile` 定数 | 名前・役職・キャッチコピーの分け方 |
| `timeline` 配列 | 経歴を「年・タイトル・所属・詳細」の構造で持つ書き方 |
| `works` 配列 | 作品を「タイトル・説明・タグ・成果指標」で持つ書き方 |
| `skills` 配列 | スキルを「カテゴリ別」にまとめる書き方 |
| `learning` 配列 | 学習中の項目に「進捗％」を持たせる書き方 |

⭐ **データを別ファイルに分離する**＝後で書き換えやすい。これがプロが習慣にしている分け方。

---

### 2️⃣ 次に `src/App.tsx`（10分）

**ここがページの組み立て役。** profile.ts のデータを取り込んで、画面に並べています。

注目してほしい点:

| 何を見る | コードのどこ |
|---|---|
| **画面全体の構造** | 上部 Header → 2カラム（左 aside + 右 main） → Footer |
| **2カラムレイアウト** | `lg:grid-cols-[320px_1fr]` で PC は左固定320px・右可変 |
| **モバイル対応** | `grid-cols-1 lg:grid-cols-[320px_1fr]` で 375px は縦積み |
| **左サイドが追従** | `lg:sticky lg:top-8` でスクロールしても左が見えたまま |
| **Card の使い分け** | ProfileCard / ContactCard / SkillsCard / AboutCard / TimelineCard ... 7つの Card |
| **Badge の使い分け** | `variant="default"` 強調 / `variant="outline"` 枠線 / `variant="success"` 緑 |
| **タイムラインの縦線** | `border-l` と絶対配置の小さい丸 (`absolute -left-` で線にドットを乗せる) |
| **作品カードのホバー** | `hover:border-blue-300 hover:shadow-sm transition-all` で動きが出る |
| **学習プログレスバー** | `<div style={{ width: '65%' }} />` でグラデの帯を出す |

⭐ 「**JSX = HTML + データの差し込み**」という感覚が掴めれば、これでReactは8割理解したと言っていい状態です。

---

### 3️⃣ 余裕があれば `package.json`（2分）

`"dependencies"` の中を眺めてください。**「このアプリで使ったライブラリ一覧」** です。

| ライブラリ | 何をしてくれる人？ |
|---|---|
| `react` | 画面を組み立てる本体 |
| `react-dom` | React と ブラウザをつなぐ |
| `lucide-react` | アイコン提供（Briefcase, Mail, Globe...） |
| `class-variance-authority` | Badge の variant 切り替えの仕組み |
| `clsx` `tailwind-merge` | クラス名を合成する `cn()` 用 |
| `@radix-ui/react-slot` | shadcn/ui の内部部品 |

⭐ **覚えなくて大丈夫。** 「AI がこういう部品を組み合わせてくれた」と分かれば十分です。

---

### 4️⃣ ⭐ 大事: `.gitignore`（1分）

**`.env` と `.env.*` の行**を探してください。これが **「.env を GitHub に上げないための設定」** です。

Stage 5 で「.env が GitHub にあがっていないか確認」したのは、この `.gitignore` が効いているからです。
